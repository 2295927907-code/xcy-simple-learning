# 光谱匹配分析器安装包工作目录

本目录用于集中保存当前版本安装包相关内容，方便后续用 PyInstaller、Inno Setup 等工具生成 Windows `Setup.exe`。

## 目录结构

- `源文件/`：当前版本程序运行所需文件副本。
- `配置/`：PyInstaller spec、Inno Setup `.iss` 等打包配置。
- `输出/`：最终生成的安装包，例如 `光谱匹配分析器_Setup.exe`。
- `临时构建/`：打包过程产生的虚拟环境、中间文件和 PyInstaller 输出。
- `build_installer.ps1`：一键构建脚本。

## 当前注意事项

- `assets/` 中包含程序和安装包可复用的图标资源：`spectrum-match-icon.ico`、`spectrum-match-icon.png`、`spectrum-match-icon-source.png`。
- 本目录只是打包工作区，不替代项目源码目录。
- 不要把用户测试数据、导出结果、`.git/`、`.npm-cache/`、`__pycache__/`、`video_spd_intro/` 或历史参数包放进安装包源文件目录。
- 构建脚本会在 `临时构建/venv` 中安装 PyInstaller 和 Python 依赖，不会修改用户电脑已有 Python。
- 构建脚本不会自动安装 Inno Setup；如果找不到 `ISCC.exe`，会先生成 PyInstaller 程序目录并提示安装 Inno Setup 6。
- Inno Setup 可以安装到所有用户的 `Program Files`，也可以安装到当前用户的 `%LOCALAPPDATA%\Programs\Inno Setup 6`；构建脚本会自动查找这两类位置。
- 当前安装向导使用 Inno Setup 默认语言文件；如果后续需要简体中文安装向导，可补充 `ChineseSimplified.isl` 后再修改 `配置/installer.iss`。

## 构建命令

在项目根目录运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\安装包\build_installer.ps1
```

如果只想先生成 PyInstaller 程序目录，不生成安装器：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\安装包\build_installer.ps1 -SkipInstaller
```

如果 PyInstaller 程序目录已经存在，只想复用它生成安装器：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\安装包\build_installer.ps1 -SkipPyInstaller
```

## 输出位置

- PyInstaller 程序目录：`安装包/临时构建/dist/光谱匹配分析器/`
- 最终外发安装器：`安装包/输出/光谱匹配分析器_Setup.exe`
- 对外分发时只发送最终外发安装器；安装器内已包含 Python 运行时、依赖包、默认光谱数据、默认 SR 文件和图标资源。

## GitHub 发布

仓库包含手动触发的 GitHub Actions 工作流：`.github/workflows/build-installer.yml`。在 GitHub 的 Actions 页面运行 `Build Windows Installer` 后，GitHub 会在 Windows runner 上构建安装器，并把 `光谱匹配分析器_Setup.exe` 发布到 Release 附件中，用户只需要下载这个文件。

## 常见失败原因

- 找不到 Python：请安装 Python 3.10+，或在 Codex 环境中运行脚本。
- 找不到 Inno Setup：请安装 Inno Setup 6，支持安装到当前用户或所有用户。
- 依赖下载失败：检查网络或 Python 包镜像设置，然后重新运行脚本。
- 程序自检失败：先回到项目根目录运行源码态自检，确认默认光谱数据和依赖可用。
